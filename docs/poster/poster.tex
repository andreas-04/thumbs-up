  \pillhead{Value Proposition}\vspace{18pt}\par

    No lightweight solution combines certificate-based authentication, fine-grained access control, and zero-config discovery in one device.\par}
\end{textblock}

% -- Objective --
\begin{textblock}{0.295}(0.02, 0.265)
  \setlength{\baselineskip}{28pt}\setlength{\parskip}{6pt}%
  \pillhead{Objective}\vspace{18pt}\par
  {\fontsize{28}{36}\selectfont\setlength{\baselineskip}{38pt}%
    Build a portable, self-hosted file server on a Raspberry Pi that provides secure
    multi-user file sharing with mTLS authentication, three-tier permissions, and mDNS
    discovery, deployable in any environment without cloud infrastructure or client
    software beyond a browser.\par}
\end{textblock}

% -- Problem Definition --
\begin{textblock}{0.295}(0.02, 0.41)
  \setlength{\baselineskip}{28pt}\setlength{\parskip}{6pt}%
  \pillhead{Problem Definition}\vspace{18pt}\par
  {\fontsize{28}{36}\selectfont\setlength{\baselineskip}{38pt}%
    Pocket-sized, self-contained file storage with certificate-based access. Plug in, sign up, and your certificate arrives by email. Unplug to take your data. No cloud, no residue.\par}
\end{textblock}

% -- guest image --
\begin{textblock}{0.295}(0.02, 0.52)

\end{textblock}
\begin{textblock}{0.295}(0.02, 0.75)
  {\fontsize{24}{30}\selectfont\itshape\color{black} Users can browse shared files
  directly in the browser after installing their client certificate.\par}
\end{textblock}

\begin{textblock}{0.295}(0.02, 0.778)
  \setlength{\baselineskip}{28pt}\setlength{\parskip}{6pt}%
  \pillhead{Key Requirements}\vspace{18pt}\par
  {\fontsize{28}{36}\selectfont\setlength{\baselineskip}{38pt}%
    All traffic uses HTTPS; regular users authenticate via mTLS. The full certificate
    lifecycle is managed on-device. A three-tier permission model covers domain, group, and per-user overrides. mDNS
    with Wi-Fi AP fallback enables zero-config discovery. All security events are
    logged and the system runs fully self-contained.\par}
\end{textblock}

% -- Concept Development --
\begin{textblock}{0.31}(0.352, 0.80)
  \setlength{\baselineskip}{28pt}\setlength{\parskip}{6pt}%
  \pillhead{Concept Development}\vspace{18pt}\par
  {\fontsize{28}{36}\selectfont\setlength{\baselineskip}{38pt}%
Two prior designs were evaluated: an SMB-over-mTLS approach requiring client-side Docker, and a QR code JWT approach lacking mutual TLS and per-user access control. The final design resolves both by delegating mTLS verification to Nginx, requiring only a browser with an installed certificate.\par}
\end{textblock}

\begin{textblock}{0.29}(0.684, 0.17)

  \pillhead{Validation}\vspace{18pt}\par
  {\fontsize{28}{36}\selectfont\setlength{\baselineskip}{38pt}%
    The backend is verified by 105 unit tests across authentication, permission resolution,
    certificate lifecycle, and path traversal prevention. The frontend is enforced through
    ESLint, TypeScript strict mode, and Vite builds. A three-workflow GitHub Actions
    pipeline runs linting, type checking, all backend tests, and Docker builds on every push.\par}
\end{textblock}
